# Idea Mix

## CONTEXT

### 適用範囲

能力カードの「成功原理」を多段推論で深く抽出し、その原理を摩擦・刺激カードと
認知的隔離（DEP-8）+ 衝突合成（DEP-9）で組み合わせ、まだ存在しない概念を生成する。

全フェーズで**マルチターンの深い思考**をサブエージェントが自律的に行う。
1ターンの浅い推論は一切行わない。

### v2 の設計思想

v1 の問題: Lens エージェントが1ターンで200字の結論を出し、それを「衝突」と呼んでいた。
本質的に1つのプロンプトで全部考えるのと変わらず、深さが不足していた。

v2 の解決:
1. **原理抽出フェーズ**を追加: 能力カードの「なぜうまくいくか」を実物を読んで多段推論で抽出
2. **Lens の深い思考**: 各 Lens が複数ラウンドの推論+自己批判+WebSearch を行う
3. **合成の深い思考**: 合成者も多段推論+WebSearch で既存性チェックを行う
4. **原理ベースの転写**: カードの表面ではなく原理レベルで接続するから、転写が深い

### 用語

| 用語 | 定義 |
|------|------|
| 能力カード | ユーザーが実際に作れる/できることの記述 |
| 成功原理 | 能力カードの「なぜうまくいくか」の本質的な構造。転写の単位 |
| 摩擦カード | 日常で感じる不便・非効率・構造的問題 |
| 刺激カード | 他領域の面白い構造・パターン・トレンド |
| Lens | 部分情報+単一思考フレームで多段推論する隔離エージェント |
| 衝突合成 | 隔離された Lens の洞察を衝突させ新概念を生む操作 |

### 前提条件

- `ideas/idea-mix/deck/capabilities.md` が存在すること
- `ideas/idea-mix/deck/frictions.md` にエントリがあること
- `ideas/idea-mix/deck/stimuli.md` にエントリがあること

| DEP-1 | 過去の出力との重複回避 | diversity-context-loader |

## ROLE

| 属性 | 内容 |
|------|------|
| **専門性** | 深い思考のオーケストレーション、認知的多様性の構造的確保 |
| **権限** | カード選択、原理抽出の起動、Lens構成決定、合成結果の記録 |
| **責任** | 全フェーズで深い思考が行われることの保証 |
| **禁止事項** | 1ターンの浅い推論で結論を出すこと |

## INTENT

### Goal

ユーザーの能力カードから成功原理を深く抽出し、その原理を摩擦・刺激と
構造的に衝突させ、まだ言語化されていない概念を生成する。

### Value

- **V-1**: 原理レベルの接続による深いアイデア（表面的な組み合わせではない）
- **V-2**: マルチターン推論による1ショットでは到達できない洞察
- **V-3**: WebSearch による新規性の裏付け

### Success Criteria

- [ ] 原理抽出で3-5個の非自明な原理が出ている
- [ ] 各 Lens が複数ラウンドの推論を経ている
- [ ] 生成された概念の新規性が WebSearch で検証されている
- [ ] ユーザーが「これは面白い」と感じる概念が含まれる

## STEPS

### 実行フロー

```
Phase 1: デッキ準備 (自動)
    ↓
Phase 2: 能力カード選択 + ユーザー確認
    ↓
Phase 3: 原理抽出 ← NEW: マルチターン深い思考（principle-extractor）
    ↓
Phase 4: 認知的隔離実行 ← UPGRADED: 原理ベース + Lens が多段推論
    ↓
Phase 5: 衝突合成 ← UPGRADED: 合成者が多段推論 + WebSearch 検証
    ↓
Phase 6: 対話的深掘り (ユーザーと対話)
    ↓
Phase 7: 記録 (自動)
```

---

### Phase 1: デッキ準備

```
1-1. 全デッキの読み込み
     - Read: ideas/idea-mix/deck/capabilities.md
     - Read: ideas/idea-mix/deck/frictions.md
     - Read: ideas/idea-mix/deck/stimuli.md

1-2. DEP-1: 過去出力チェック（オプション）
     - ideas/idea-mix/ に過去出力がある場合:
       diversity-context-loader を実行
         output_dir: ideas/idea-mix/
         lookback_count: 4
         extract_field: "## 生成概念" セクション
     - 初回実行の場合はスキップ
```

### Phase 2: 能力カード選択

```
2-1. 能力カードから1枚を選択
     - DEP-2 のローテーションまたはユーザー指定
     - ユーザーに選択結果を提示:

     ---
     今回の能力カード:
       concrete: {具体的な記述}
       abstract: {抽象化された記述}

     このカードの「成功原理」を深く分析します。
     別のカードにしたい場合は指示してください。
     ---

2-2. 摩擦・刺激カードのプリセレクション
     - 摩擦カードから10-15枚をランダム+ローテーションで選択
     - 刺激カードから5-10枚をランダム+ローテーションで選択
     - （この時点ではユーザーに提示しない。Phase 4 で Lens に渡す用）
```

### Phase 3: 原理抽出（マルチターン深い思考）

**ここが idea-mix v2 の核心。能力カード1枚の「なぜうまくいくか」を深く分析する。**

```
3-1. principle-extractor エージェントを起動

     Agent(
       name: "principle-extractor",
       model: opus,
       prompt: """
         あなたは能力カードの「成功原理」を抽出するアナリストです。
         principle-extractor.md のプロトコルに従って多段推論を行ってください。

         対象能力カード:
           concrete: {concrete}
           abstract: {abstract}

         プロジェクトのルートディレクトリ: {project_root}

         重要:
         - 必ず実物のファイルを読んでください（.claude/skills/, .claude/rules/ 等）
         - git log でこの能力の進化を追ってください
         - 仮説→批判→精製のループを必ず回してください
         - 「モジュール化が良い」のような自明な原理は返さないでください
         - 各原理に転写可能な他のドメイン3-5個を必ず含めてください
       """
     )

3-2. 結果の受け取り
     - 3-5個の原理（各原理に定式化・根拠・反例・転写可能領域）
     - 棄却された仮説のリスト

3-3. ユーザーに原理を提示
     ---
     抽出された成功原理:

     1. {原理名}: {定式化}
        → 転写可能: {領域1}, {領域2}, {領域3}

     2. {原理名}: {定式化}
        → 転写可能: {領域1}, {領域2}, {領域3}

     3. {原理名}: {定式化}
        → 転写可能: {領域1}, {領域2}, {領域3}

     これらの原理を摩擦・刺激カードと衝突させます。
     ---
```

### Phase 4: 認知的隔離実行（DEP-8 v2）

**各 Lens が多段推論を行う。1ターンの結論生成ではない。**

```
4-1. Lens 構成の決定
     Phase 3 の原理数に応じて Lens を構成:

     Lens A: 原理1 × 摩擦カード10枚 + constrain（制約思考）
       → 「この原理で解決できる摩擦を見つけ、転写方法を深く考えよ」
       → 原理1の情報のみ。原理2,3は隠蔽。刺激カードも隠蔽。

     Lens B: 原理2 × 刺激カード5枚 + analogize（類推思考）
       → 「この原理と構造的に類似するパターンを刺激から見つけ、新応用を考えよ」
       → 原理2の情報のみ。原理1,3は隠蔽。摩擦カードも隠蔽。

     Lens C: 摩擦カード5枚のみ + deconstruct（分解思考）
       → 「これらの摩擦の根本原因を徹底的に掘り下げよ。解決策は考えるな」
       → 能力カード・原理・刺激は一切隠蔽。

     Lens D: 刺激カード5枚のみ + reframe（再定義思考）
       → 「これらの刺激が示すパターンから、普通は問わない問いを立てよ」
       → 能力カード・原理・摩擦は一切隠蔽。

4-2. cognitive-isolation v2 シーケンスを実行
     - 4つの Lens を並列起動（Agent tool × 4、各 Sonnet）
     - 各 Lens は内部で多段推論を実行:
       * 情報の深い理解
       * 思考フレームに従った複数ラウンドの分析
       * 自己批判
       * WebSearch による調査（必要に応じて）
       * 洞察の精製
     - 各 Lens が返す形式: 洞察 + 到達経路 + 開く可能性

4-3. 洞察を収集
     - 4つの深い洞察（各 Lens から1つ）を取得
     - 合成者には洞察の3要素のみ渡す（推論の詳細過程は渡さない）
```

### Phase 5: 衝突合成（DEP-9 v2）

**合成者も多段推論を行い、WebSearch で新規性を検証する。**

```
5-1. 合成モードの決定
     ISO 週番号 % 3:
       0 → emergent（矛盾からの創発）
       1 → bridge（隠れた共通構造）
       2 → invert（暗黙の前提の反転）

5-2. collision-synthesis v2 シーケンスを実行
     - collision-synthesizer エージェント（Opus）を起動
     - 4つの洞察を入力
     - 合成者は内部で多段推論を実行:
       * 各洞察の構造分析
       * 合成モードに応じた複数ラウンドの思考
       * WebSearch による既存性チェック
       * 「なぜ今まで存在しなかったか」の仮説生成
       * 最小実験の設計
     - concept_count: 3-5

5-3. 生成された概念をユーザーに提示
     ---
     ## 衝突から生まれた概念

     ### 1. {概念名}
     {説明（3-5文）}
     💥 衝突元: 洞察{A} × 洞察{B}
     🆕 新規性: {なぜ新しいか + WebSearch結果}
     🧪 最小実験: {1日でできる検証方法}

     ### 2. {概念名}
     ...

     ### 3. {概念名}
     ...

     ### 却下された概念
     - {概念名}: {WebSearch で既存と判明 or 折衷に過ぎないため却下}

     どれが面白いですか？ 深掘りしたい番号を教えてください。
     （複数可。「全部面白くない」も OK — カードを変えて再挑戦します）
     ---
```

### Phase 6: 対話的深掘り

```
6-1. ユーザーの選択を受け取る
     - 番号指定 → 選択された概念を深掘り
     - 「全部面白くない」→ Phase 2 に戻り、別の能力カードで再実行
     - 「もっと出して」→ Phase 5 を別の synthesis_mode で再実行

6-2. deepener エージェントの思考法で対話を進める

     Step A: 存在チェック
       「この概念は既に〜として存在していませんか？」
       → WebSearch 結果も踏まえつつ、ユーザーとの対話で差別化を明確化

     Step B: Devil's Advocate
       「もしこの概念が機能しないとしたら、最大の理由は？」
       → 3つの反論を提示し、ユーザーの見解を求める

     Step C: 本質の蒸留
       「この概念の核心を一言で言うと？」
       → 対話を通じて不要な要素を削ぎ落とす

     Step D: 最小実験の具体化
       「明日からこれを検証するとしたら？」
       → ユーザーの手持ちのカード（能力）を踏まえた具体的な検証方法

6-3. 深掘り結果を構造化
```

### Phase 7: 記録

```
7-1. ファイル生成
     Write: ideas/idea-mix/YYYY-MM-DD-{slug}.md
     テンプレート: plugins/idea-mix/templates/output.md に従う

7-2. フロントマター:
     ---
     title: "{概念名}"
     created: YYYY-MM-DD
     tags: [idea-mix]
     status: raw
     source:
       capability_card: "{使用した能力カード}"
       principles_used: ["{原理1}", "{原理2}"]
       frictions_used: ["{摩擦1}", "{摩擦2}"]
       stimuli_used: ["{刺激1}"]
     collision:
       lens_frames: ["{frame1}", "{frame2}", "{frame3}", "{frame4}"]
       synthesis_mode: "{emergent|bridge|invert}"
     novelty_score: {N}
     ---

7-3. Git コミット
     種別: ideas
     メッセージ: "ideas: idea-mix — {概念名}"
```

## PROOF

### Verification

- [ ] principle-extractor が実物のファイルを読んで多段推論したか
- [ ] 抽出された原理が「自明でない」か（「モジュール化が良い」のような原理はNG）
- [ ] 各 Lens が複数ラウンドの推論を行ったか（1ターン完結ではないか）
- [ ] 各 Lens が部分情報のみ受け取っているか（認知的隔離が保たれているか）
- [ ] collision-synthesizer が WebSearch で既存性チェックを行ったか
- [ ] 「なぜ今まで存在しなかったか」の説明があるか

### Done Criteria

- [ ] ideas/idea-mix/YYYY-MM-DD-{slug}.md が生成された
- [ ] 概念が原理レベルで接続されている（表面的な組み合わせではない）
- [ ] 新規性が WebSearch で裏付けられている
- [ ] 最小実験が具体的に定義されている

### Do / Don't

**Do**:
- ✅ 全サブエージェントにマルチターン推論を指示する
- ✅ principle-extractor に実物のファイルを必ず読ませる
- ✅ 各 Lens に異なる部分情報を渡す（認知的隔離の徹底）
- ✅ collision-synthesizer に WebSearch で新規性検証をさせる
- ✅ ユーザーのカード変更要望を柔軟に受け入れる
- ✅ 「面白くない」結果が出たらカードを変えて再挑戦する

**Don't**:
- ❌ どのフェーズでも1ターンの浅い推論で終わらせる
- ❌ カードの表面的な記述だけで原理を抽出する（実物を読め）
- ❌ 全情報を1つのプロンプトに入れて考えさせる
- ❌ WebSearch なしで「新しい」と主張する
- ❌ 推論過程を合成者に見せる（洞察の3要素のみ）
- ❌ 「面白そうですね」で深掘りを省略する
