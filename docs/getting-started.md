# Getting Started

ideate は2つのプラグインを持つアイデア発想基盤。
それぞれ目的が異なるので、やりたいことに応じて使い分ける。

## どっちを使えばいい？

| やりたいこと | 使うプラグイン |
|-------------|--------------|
| アプリ・サービスのアイデアが欲しい | **app-discovery** |
| もっと自由に「こんなことできるんじゃ？」を考えたい | **idea-mix** |
| 日常の不便を記録しておきたい | **app-discovery** (`/pain-log`) |
| 自分の能力カードを整理したい | **idea-mix** (`/idea-mix-deck`) |

**両方使うのが理想的。** pain-log で溜めた観察は idea-mix の摩擦カードにも使える。
seed-hunt で拾ったトレンドは idea-mix の刺激カードにもなる。

---

## app-discovery — アプリアイデアを見つける

### 概要

ペインポイント（日常の不便）を起点に、アプリ・サービスのアイデアを発見→検証→設計するパイプライン。

```
/pain-log → /seed-hunt → /seed-validate → /seed-shape → /pipeline-pulse
  観察      ネット収集     市場検証        機能設計      週次レビュー
```

### 日常の使い方

#### 毎日: 不便を記録する（30秒）

```
/pain-log "電車で読みたい記事をあとで読むのが面倒"
```

引数なしで対話モードも可:
```
/pain-log
→ 「何が不便でしたか？」
→ 回答すると構造化して記録
```

- 1日に何回でもOK
- ネット検索はしない（自分の一次体験だけ）
- `ideas/pain-log/YYYY-MM-DD.md` に蓄積される

#### 毎週: アイデアの種を収集する

```
/seed-hunt
```

- 完全自動。Reddit・X・HN・はてブなどから収集
- 6軸スコアリング（Pain/Frequency/Market/Gap/Fit/Novelty）
- Featured 3件 + Most Novel 1件を選出
- `ideas/app-seeds/YYYY-MM-DD.md` に出力

#### 気になるアイデアを検証する

```
/seed-validate
```

- seed-hunt の結果から3-4個を選んで深掘り検証
- 競合調査・需要エビデンス・市場シグナルを収集
- 5軸スコアで比較し、推奨1件 + MVP定義を出力

#### アイデアを具体化する

```
/seed-shape
```

- 検証済みアイデアを機能仕様に落とす
- JTBD・Shape Up・行動設計などのフレームワークを適用
- ワークショップ形式で対話しながら設計

#### 週末: パイプラインを俯瞰する

```
/pipeline-pulse
```

- pain-log → seed → validation → shaping の全体を俯瞰
- P.A.I.N.フレームワークで摩擦をクラスタリング
- Novelty Health（新規性の健全性）をチェック
- 次のアクションを提案

---

## idea-mix — 自由にアイデアを発想する

### 概要

能力・摩擦・刺激の3種類のカードを「認知的に隔離された」複数の視点で分析し、
その結論を衝突させることで、まだ存在しない概念を生み出す。

```
カードを引く → 4つのLensが独立に考える → 結論を衝突させる → 新概念 → 対話で研磨
```

**app-discovery との違い**: app-discovery は「アプリアイデア」に特化。
idea-mix は「何でもいいから面白いことを思いつきたい」時に使う。

### 始め方

#### 1. まず能力カードを整理する

```
/idea-mix-deck list
```

初期デッキに8枚の能力カードが入っている。
足りなければ追加:

```
/idea-mix-deck add "Figmaでデザインができる"
```

自動で抽象化される:
- concrete: Figmaでデザインができる
- abstract: ビジュアルプロトタイピングツールでインタラクション設計ができる

#### 2. 摩擦・刺激カードを追加する

手動で追加:
```
/idea-mix-deck add "毎回同じ会議で同じことを話している"
/idea-mix-deck add "GitHubのCopilot Workspaceがissueから実装まで自動化し始めた"
```

pain-log や daily-trend から自動同期:
```
/idea-mix-deck sync
```

#### 3. アイデアを生成する

```
/idea-mix
```

フローは自動で進む:

1. **デッキからカードが選ばれる**（変更可能）
   ```
   今回のカード:
   🃏 能力: 手続き的知識を宣言的に定義しAIランタイムに実行させる基盤構築能力
   🔥 摩擦: 毎回同じ会議で同じことを話している
   ⚡ 刺激: Copilot Workspaceがissueから実装まで自動化

   このカードで進めますか？
   ```

2. **4つのLensが独立に考える**（自動・並列）
   - 各Lensは部分情報しか見えない
   - 各Lensは異なる思考法で考える
   - 互いの存在を知らない

3. **結論が衝突する**（自動）
   - 4つの結論だけを見て、新概念を生成
   - 「なぜ今まで存在しなかったか」を必ず説明

4. **面白いものを選んで深掘り**（対話）
   ```
   どれが面白いですか？
   ```
   - 選んだ概念に対して反論される
   - 「これ既にあるんじゃ？」を問われる
   - 最小実験に落とし込む

5. **記録される**
   - `ideas/idea-mix/YYYY-MM-DD-{slug}.md` に保存

#### 4. 振り返る

```
/idea-mix-review
```

- どのLens構成が面白い結果を出したか
- どのカード組み合わせが有効だったか
- Novelty Healthの推移

### なぜ「認知的隔離」が必要なのか

普通に「この3つのカードでアイデアを考えて」とAIに聞くと、
毎回似たような結論に着地する。これは:

1. 最初に「これはこういう問題だ」とフレーミングした瞬間、後続の思考が全部汚染される
2. 全情報が見えると「当たり前の結論」しか出ない
3. 同じ推論エンジンが同じ方向に考える

idea-mix はこれをアーキテクチャで解決する:
- 各Lensに**部分情報だけ**渡す → 解決手段に引っ張られない
- 各Lensに**異なる思考法**を強制する → 同じ情報でも違う結論
- 合成者は**結論だけ**見る → 推論バイアスが伝搬しない

## 両方を組み合わせた週間ルーティン

| タイミング | やること | コマンド |
|-----------|---------|---------|
| 毎日（随時） | 不便を感じたら記録 | `/pain-log "..."` |
| 週1回 | アプリアイデアを収集 | `/seed-hunt` |
| 週1回 | 自由にアイデア発想 | `/idea-mix` |
| 週末 | パイプライン俯瞰 | `/pipeline-pulse` |
| 月1回 | idea-mix振り返り | `/idea-mix-review` |
| 随時 | 気になるアイデアを検証 | `/seed-validate` → `/seed-shape` |
| 随時 | カードを追加 | `/idea-mix-deck add "..."` |

### データの流れ

```
/pain-log で記録した観察
  ├─→ /seed-hunt の検索ヒントに使われる
  └─→ /idea-mix-deck sync で摩擦カードになる

/seed-hunt で拾ったトレンド
  └─→ /idea-mix-deck sync で刺激カードになる

/idea-mix で生成した概念
  ├─→ 面白ければ /seed-validate で市場検証
  └─→ さらに進めば /seed-shape で機能設計
```
